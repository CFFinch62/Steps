Getting Started with Steps

Install and start teaching programming in less than 5 minutes

Prerequisites

Before you begin, make sure you have:

  • Python 3.10 or higher installed
  • pip (Python package manager)
  • A terminal or command prompt
  • A text editor or IDE

Don't have Python? Download it from python.org

Installation

1

Clone the Repository

Download the Steps source code from GitHub:

git clone <repository-url>
cd Steps

Replace <repository-url> with the actual GitHub repository URL.

2

Create Virtual Environment (Recommended)

Keep Steps isolated from other Python projects:

# Create virtual environment
python3 -m venv venv

# Activate it
source venv/bin/activate          # On Linux/Mac
venv\Scripts\activate            # On Windows

You'll need to activate the virtual environment each time you use Steps.

3

Install Steps

Install Steps in development mode:

pip install -e .

The -e flag installs in editable mode, perfect for exploring and customizing.

4

Install IDE Dependencies (Optional)

For the best experience, install the Steps IDE:

pip install textual watchfiles

The IDE provides a graphical development environment built specifically for Steps.

5

Verify Installation

Test that Steps is installed correctly:

# Try the REPL
python -m steps_repl.main

# You should see:
Steps REPL v0.1 - Educational Programming Environment
Type 'help' for available commands, 'exit' to quit.

>>>

Type exit to quit the REPL.

Your First Steps Program

Let's create a simple program to make sure everything works!

1

Create Project Folder

mkdir hello_world
cd hello_world
2

Create the Building File

Create a file named hello_world.building with this content:

building: hello_world
    note: My first Steps program!

    display "Hello, Steps!"
    display "Welcome to programming!"

    exit
3

Run Your Program

python -m steps.main run .

You should see:

Hello, Steps!
Welcome to programming!

Congratulations! You've just run your first Steps program!

Development Tools

Steps comes with three tools for different learning scenarios:

üñ•Ô∏è Command Line

Professional workflow for running and checking programs.

Common Commands:

# Run a program
python -m steps.main run <path>

# Check syntax without running
python -m steps.main check <path>

# Generate program diagram
python -m steps.main diagram <path>

Best for: Teaching command-line skills, automated testing, CI/CD integration

üí¨ Interactive REPL

Experiment and explore concepts without creating files.

How to Use:

# Start the REPL
python -m steps_repl.main

# Try commands:
>>> set name to "Alice"
>>> display name
Alice
>>> vars
Variables:
  name = "Alice"

Best for: Interactive demonstrations, exploring new concepts, quick experiments

üé® Steps IDE

Full development environment with project browser and integrated terminal.

Launch:

python -m steps_ide.main

Key Shortcuts:

  • F5 - Run project
  • F6 - Check syntax
  • Ctrl+O - Open file
  • Ctrl+S - Save file
  • Ctrl+Shift+P - Project browser

Best for: Complete project development, visual learners, classroom demos

Tips for Educators

üéØ Start Simple

Begin with the REPL and single-file buildings. Don't introduce floors and steps until students are comfortable with basic syntax.

üìä Show the Structure

Use the file browser to show how program organization is visible. This is one of Steps' key teaching advantages.

üîÑ Use the Tutorial

The included tutorial is classroom-ready. Each lesson is self-contained and builds progressively.

üí° Encourage Experimentation

The REPL is perfect for "what if" questions. Students can try ideas immediately without creating files.

üêõ Embrace Errors

Steps error messages are educational. When students get errors, use them as teaching moments.

üèóÔ∏è Emphasize the Metaphor

The building metaphor isn't just cute - it's pedagogically powerful. Reinforce it consistently.

Troubleshooting

Python not found

Problem: python: command not found

Solution: Try python3 instead of python, or install Python from python.org

Module not found

Problem: ModuleNotFoundError: No module named 'steps'

Solution: Make sure you've run pip install -e . from the Steps directory and activated your virtual environment.

Permission denied

Problem: Permission errors when installing

Solution: Use a virtual environment or add --user flag: pip install --user -e .

IDE won't start

Problem: Error launching Steps IDE

Solution: Make sure you've installed IDE dependencies: pip install textual watchfiles
