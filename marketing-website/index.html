The Building Metaphor

Steps uses architecture to make program structure tangible

Building
Complete program (entry point)
.building
‚Üì
Floor
Functional grouping of related steps
.floor
‚Üì
Step
Single unit of work (one file per step)
.step
‚Üì
Riser
Private helper function within a step
(inside .step)

Just Like a Real Building

  • Buildings are complete, standalone structures - your whole program
  • Floors group related functionality - like departments in an office building
  • Steps are individual units of work - one clear task per file
  • Risers are hidden helpers - internal scaffolding that supports steps

This hierarchy enforces decomposition naturally - you can't write monolithic code in Steps!

Clear, Readable Syntax

Code that reads like English instructions

Variables & Data

set name to "Alice"
set age to 25
set is_student to true
set scores to [95, 87, 92]

Conditions

if age is greater than 18
    display "Adult"
otherwise if age is greater than 13
    display "Teen"
otherwise
    display "Child"

Loops

repeat 5 times
    display "Hello!"

for each score in scores
    display score

Functions (Steps)

step: calculate_tax
    expects: amount, rate
    returns: tax

    do:
        set tax to amount * (rate / 100)
        return tax
