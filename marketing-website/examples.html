Example 2: Tip Calculator

Intermediate

A practical application that calculates tips and totals. Demonstrates steps, floors, and parameter passing.

Concepts: User input, steps with parameters, floors, function calls, number formatting

Project Structure

tip_calculator/
‚îú‚îÄ‚îÄ tip_calculator.building
‚îî‚îÄ‚îÄ math/
    ‚îú‚îÄ‚îÄ math.floor
    ‚îú‚îÄ‚îÄ calculate_tip.step
    ‚îî‚îÄ‚îÄ round_to_cents.step

tip_calculator.building

building: tip_calculator
    note: Calculate tip and total for a restaurant bill

    display "Welcome to Tip Calculator!"
    display ""

    display "Enter bill amount: $"
    set bill_amount to input as number

    display "Enter tip percentage (e.g., 15, 18, 20): "
    set tip_percent to input as number

    call calculate_tip with bill_amount, tip_percent
        storing result in tip_amount

    set total to bill_amount + tip_amount

    display ""
    display "Bill:  $" added to (bill_amount as text)
    display "Tip:   $" added to (tip_amount as text)
    display "Total: $" added to (total as text)

    exit

math/calculate_tip.step

step: calculate_tip
    belongs to: math
    expects: amount, percent
    returns: tip

    note: Calculate tip based on percentage

    riser: calculate_percentage
        expects: value, pct
        returns: result

        do:
            set result to value * (pct / 100)
            return result

    declare:
        raw_tip as number

    do:
        call calculate_percentage with amount, percent
            storing result in raw_tip
        call round_to_cents with raw_tip
            storing result in tip
        return tip

How to Run

python -m steps.main run projects/tip_calculator/

Sample Session:

Welcome to Tip Calculator!

Enter bill amount: $50.00
Enter tip percentage (e.g., 15, 18, 20): 18

Bill:  $50.0
Tip:   $9.0
Total: $59.0

Key Takeaways

  • Steps can accept parameters with expects:
  • Steps can return values with returns:
  • Risers are private helpers within a step
  • Floors organize related steps together
  • Use input as number to get numeric input

Example 3: Price Calculator with Discount

Intermediate

Calculate prices with quantity and discounts. Shows multiple steps working together.

Concepts: Multiple steps, parameter passing, mathematical operations, floor organization

Project Structure

price_calculator/
‚îú‚îÄ‚îÄ price_calculator.building
‚îî‚îÄ‚îÄ calculations/
    ‚îú‚îÄ‚îÄ calculations.floor
    ‚îú‚îÄ‚îÄ calculate_subtotal.step
    ‚îî‚îÄ‚îÄ apply_discount.step

calculations/calculate_subtotal.step

step: calculate_subtotal
    belongs to: calculations
    expects: price, quantity
    returns: total

    do:
        set total to price * quantity
        return total

calculations/apply_discount.step

step: apply_discount
    belongs to: calculations
    expects: amount, percent
    returns: discounted

    declare:
        discount as number

    do:
        set discount to amount * (percent / 100)
        set discounted to amount - discount
        return discounted

Key Takeaways

  • Steps should be small and focused on one task
  • Chain steps together to build complex functionality
  • Use meaningful names that describe what the step does
  • Group related steps in the same floor

Example 4: Loops and Repetition

Beginner

From the tutorial - demonstrates all three types of loops in Steps.

Concepts: repeat times, while loops, for each loops, counters, lists

Repeat a Fixed Number of Times

set counter to 1
repeat 5 times
    display (counter as text)
    set counter to counter + 1

While Loop with Condition

set countdown to 5
while countdown is greater than 0
    display (countdown as text)
    set countdown to countdown - 1
display "Blast off!"

For Each with List

set fruits to ["apple", "banana", "cherry", "date"]
for each fruit in fruits
    display "- " added to fruit

Key Takeaways

  • Use repeat N times when you know the count
  • Use while condition for conditional looping
  • Use for each item in list to process collections
  • All loop types use clear, English-like syntax

Example 5: Working with Lists and Tables

Beginner

Store and manipulate collections of data.

Concepts: Lists, tables (dictionaries), indexing, adding/removing items

Lists

note: Create a list
set colors to ["red", "green", "blue"]

note: Access by index (0-based)
set first to colors[0]
display first  note: Shows "red"

note: Add and remove items
add "yellow" to colors
remove "green" from colors

note: Check membership
if "red" is in colors
    display "Has red!"

note: Get length
set count to length of colors
display "Number of colors: " added to (count as text)

Tables (Dictionaries)

note: Create a table with key-value pairs
set person to ["name": "Alice", "age": 25, "city": "Boston"]

note: Access values by key
set name to person["name"]
display name  note: Shows "Alice"

note: Update or add new values
set person["age"] to 26
set person["email"] to "alice@example.com"

note: Display all info
display person["name"] added to " is "
    added to (person["age"] as text)
    added to " years old"

Key Takeaways

  • Lists hold ordered collections: [item1, item2, item3]
  • Tables hold key-value pairs: ["key": value]
  • Both use square brackets for access: list[0], table["key"]
  • Lists are 0-indexed (first item is at position 0)
  • Tables are perfect for structured data like records

Example 6: Error Handling

Intermediate

Handle errors gracefully with attempt blocks.

Concepts: Error handling, attempt/unsuccessful/continue, problem_message variable

Basic Error Handling

attempt:
    display "Enter a number: "
    set num to input as number
    set result to 100 / num
    display "Result: " added to (result as text)
if unsuccessful:
    display "Error: " added to problem_message
    display "Please enter a valid number."
then continue:
    display "Operation complete."

Key Takeaways

  • attempt: block contains code that might fail
  • if unsuccessful: runs if an error occurs
  • then continue: always runs (like finally)
  • problem_message contains the error description
